# Procedure: Generate Review Bundle

Main Agent generates a review bundle after closing a TKT bundle, for PM Agent and user discussion.

## Steps

### 1. Gather Results

Read all tickets in the bundle:
```bash
bash "<this-skill-dir>/scripts/tkt.sh" status --bundle <B-NNN>
```

For each completed ticket, read its `result` section for summaries and artifacts.

### 2. Compile Review

Edit the `review.yaml` generated by the close command:

**summary**: One paragraph describing what was accomplished, what changed, and what the outcome is.

**discussion_points**: Issues or decisions that need user input:
- Trade-offs made during implementation
- Deferred work or known limitations
- Alternative approaches considered but not taken
- Quality concerns from audit

**next_actions**: Suggested follow-ups:
- Related improvements discovered during work
- Technical debt identified
- Features or fixes that naturally follow

### 3. Include Audit Results

If TKT-A00 was completed, incorporate:
- `checked_items`: What was spot-checked
- `findings`: Issues found (or confirmation of quality)
- `quality_score`: 1-5 rating

### 4. Present for Review

PM Agent presents the review to the user with:
1. Executive summary (1-2 sentences)
2. Ticket completion table
3. Key findings from audit
4. Discussion points (as questions for user)
5. Suggested next steps

The user may:
- Approve and move on
- Request changes (→ new bundle)
- Ask questions (→ PM Agent answers from context)
- Adjust roadmap priorities
